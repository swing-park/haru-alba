# 하루알바 사업계획서

> 초단기 부수입 전용 B2B2C + C2C 인력 매칭 플랫폼

---

## 1. 서비스 개요

| 항목 | 내용 |
|------|------|
| 서비스명 | 하루알바 |
| 서비스 유형 | B2B2C + C2C 복합 인력 매칭 플랫폼 |
| 핵심 타깃 | 단순 인력이 필요한 사업자(B2B) + 부수입을 원하는 개인(C) |
| 핵심 가치 | "오늘, 지금 당장 할 수 있는 알바" |
| 캐치프레이즈 | 퇴근길에 10만원 벌고 가기 |

### 서비스 한 줄 정의
> 구인·구직 모두에게 **가장 빠르고 단순한** 초단기 인력 매칭 경험을 제공하는 플랫폼

---

## 2. 문제 정의 및 시장 기회

### 2-1. 현재 시장의 문제

**구직자(C) 입장**
- 기존 알바 플랫폼(알바몬, 알바천국)은 이력서 작성, 면접, 채용 등 과정이 복잡함
- "오늘 몇 시간만 일하고 싶다"는 수요를 충족하는 전용 서비스가 없음
- 부수입 목적의 초단기 근무를 원하는 직장인, 대학생 수요는 높으나 공급이 부족

**구인자(B2B/C) 입장**
- 갑작스러운 인력 공백(직원 결근, 행사 등)을 빠르게 채울 수 있는 수단이 없음
- 당근마켓에 "바퀴벌레 잡아주시면 3만원" 같은 비정형 인력 수요가 존재하지만 플랫폼화되어 있지 않음
- 정식 채용 과정 없이 당일 또는 단기로 쓸 수 있는 인력 확보가 어려움

### 2-2. 시장 기회

- 국내 단기·일용직 근로자 규모는 연간 수백만 명 수준
- 긱 이코노미(Gig Economy) 트렌드 확산으로 단기·자유 근무 선호 증가
- 기존 경쟁사들은 **초단기 부수입 전용**으로 브랜딩된 서비스가 없음
- "퇴근 후 2~3시간", "주말 반나절" 같은 초단기 수요를 정면으로 겨냥한 서비스 부재

## 3. 서비스 구조 및 핵심 기능

### 3-1. 사용자 구조

```
[B2B 구인자 - 사업자]
      ↓ 공고 등록
  [하루알바 플랫폼]
      ↓ 매칭 및 알림
[C 구직자 - 개인]

[C 구인자 - 개인]  ←→  [C 구직자 - 개인]  (C2C)
```

### 3-2. 핵심 기능 정의

**구직자(C) 기능**
- 본인인증 기반 회원가입
- 희망 지역 / 날짜 / 시간대 / 업종 사전 설정 및 예약
- 새로운 공고 등록 시 조건 매칭 푸시 알림
- 지원 → 확정 → 출근 → 정산 플로우
- 후기 및 신뢰도 점수 적립

**구인자(B2B/C) 기능**
- 본인인증 + 사업자등록증 인증(B2B) 기반 회원가입
- 공고 등록 (날짜, 시간, 장소, 업무 내용, 급여)
- 지원자 확인 및 확정
- 후기 및 신뢰도 점수 부여

**플랫폼 공통 기능**
- 양방향 평점 및 후기 시스템
- 신뢰도 지표 (출근율, 매너점수 등)
- 정산 및 수수료 처리
- 노쇼 패널티 시스템 (고도화 단계)

## 4. 비즈니스 모델

### 4-1. 수익 구조

| 수익 유형 | 내용 | 적용 시점 |
|-----------|------|-----------|
| 거래 수수료 | 매칭 완료 시 구인/구직 양측에 수수료 부과 | MVP ~ |
| 광고 수익 | 공고 상단 노출 광고 (B2B 대상) | 유저 풀 확보 후 |
| 프리미엄 기능 | 빠른 매칭, 우선 노출 등 유료 기능 | 중기 이후 |
| 데이터 수익 | 인력 수요 트렌드 데이터 B2B 판매 | 장기 |

### 4-2. 수수료 정책 (초안)
- 구직자(C): 거래 금액의 3~5%
- 구인자(B2B/C): 거래 금액의 5~8%
- 초기에는 구직자 수수료 면제로 구직 풀 확대 유도 고려

## 5. 초기 Go-to-Market 전략

> 핵심 과제: 공급(B2B 구인자) 먼저 확보 → 수요(C 구직자) 유입 순서

### 5-1. Phase 1 — 공급 확보 (0~3개월)

- 대표자 개인 네트워크의 자영업자, 소상공인 대상으로 직접 영업
- 초기 수수료 무료 정책으로 공고 등록 유도
- 목표: 최소 공고 30건 확보 후 구직자 모집 시작

### 5-2. Phase 2 — 구직자 유입 (3~6개월)

- SNS(인스타그램, 틱톡) 기반 "퇴근길 10만원" 콘텐츠 마케팅
- 오프라인: 강남역, 홍대입구 등 유동인구 밀집 지역 배너/전단 광고
- 대학교 커뮤니티(에브리타임 등) 타깃 바이럴 마케팅

### 5-3. Phase 3 — 플랫폼 정상화 (6개월~)

- 구인·구직 풀이 일정 수준 이상 확보되면 정상 플랫폼 형태로 전환
- 수수료 과금 시작
- 광고 상품 출시

## 6. 경쟁사 분석 및 차별화 포인트

### 6-1. 경쟁사 비교

| 서비스 | 특징 | 약점 |
|--------|------|------|
| 당근알바 | 지역 기반, 높은 MAU | 단기 전용 브랜딩 없음, C2C 비정형 수요 미지원 |
| 알바몬/알바천국 | 방대한 공고량 | 초단기·당일 매칭에 최적화되지 않음 |
| 크몽/숨고 | 프리랜서 기반 | 전문직 중심, 단순 노동 수요 미충족 |

### 6-2. 하루알바 차별화 포인트

- **처음부터 초단기 부수입 전용**으로 브랜딩 — "이건 알바 앱이 아니라 부수입 앱이다"
- **비정형 C2C 수요까지 포괄** — "바퀴벌레 잡아주세요 3만원" 같은 수요도 공고화 가능
- **구직자 사전 예약 시스템** — 공고가 없어도 조건을 미리 설정해두면 매칭 알림 수신
- **초단기 특화 UX** — 이력서 없음, 면접 없음, 지원 즉시 확정 가능한 간소화된 플로우

## 7. 리스크 및 대응 방안

| 리스크 | 내용 | 대응 방안 |
|--------|------|-----------|
| 노쇼 문제 | 구직자가 확정 후 무단 불참 | 출근율 지표 공개 + 노쇼 패널티 시스템 |
| 신뢰/안전 문제 | 모르는 사람을 집/매장에 들이는 불안감 | 본인인증 필수 + 양방향 평점 + 신뢰도 점수 |
| 초기 공급 부족 | B2B 공고가 적어 구직자 이탈 | 지인 네트워크 기반 초기 공고 수동 확보 |
| 경쟁사 모방 | 대형 플랫폼이 유사 기능 출시 | 브랜드 충성도 및 초단기 UX 전문성으로 대응 |
| 수익 지연 | 초기 거래량 부족으로 수수료 수익 미미 | 자비 운영 + 빠른 MVP 검증으로 런웨이 최소화 |

## 8. 개발 로드맵

### Phase 1 — MVP (0~3개월)
> 목표: 핵심 매칭 플로우 동작 확인

- [ ] 본인인증 기반 회원가입 (구인/구직 구분)
- [ ] 구직자 희망 조건 설정 (지역, 날짜, 시간, 업종)
- [ ] 구인자 공고 등록
- [ ] 매칭 알림 발송
- [ ] 지원 → 확정 플로우
- [ ] 기술 스택: React, Next.js, TypeScript, NestJS, MongoDB, AWS

### Phase 2 — 고도화 (3~6개월)
> 목표: 신뢰 시스템 및 정산 플로우 완성

- [ ] 양방향 평점 및 후기 시스템
- [ ] 신뢰도 점수 (출근율, 매너점수)
- [ ] 정산 및 수수료 처리 시스템
- [ ] 푸시 알림 고도화

### Phase 3 — 플랫폼화 (6개월~)
> 목표: 수익 모델 가동 및 마케팅 본격화

- [ ] 광고 상품 출시 (공고 상단 노출)
- [ ] 노쇼 패널티 시스템
- [ ] 프리미엄 기능 (우선 매칭 등)
- [ ] 데이터 대시보드 (B2B 구인자용)
